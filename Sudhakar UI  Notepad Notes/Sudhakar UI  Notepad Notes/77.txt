                                Routing - Routing enables navigation in application. - Routing is a technique used in web application to configure user and SEO friendly URL's. - Routing can be implemented client side and server side. - Server side routes are used to configure request and response. - A route can handle request from client and send a customized response. 
 
Route Flow: 
- A client request is sent to web server. - A Http Handler validates the request and sends to Routing Engine. - Routing Engine verifies the requested path by comparing with the path defined in application 
Route Table. - If requested path is matching then it send a response to client request. - If path mismatch then it sends an error page with HTTP status code. 
 
Configuring Routes: - Every route comprises of a request path and callback function. 
 
Syntax: 
        app.get("path", function() { });   // path:request path  function(){}: callback () 
 - Path comprises of entities and wild cards 
 
    "/"            It executes when user request is not for any specific. 
    "*"            It executes when requested path mismatch. [not found] 
    "/name"        It executes when requested path exactly matches with given name 
    "/colou?r"        It executes when requested path is matching regular expression 
 - The callback function comprises of various objects, the primary are 
        a) request 
        b) response 
 - A request object can request content from client, which includes 
        request.query            => query string 
        request.cookies        => cookies 
        request.body            => form body 
        request.params        => route parameters 
        etc.. 
 - A response object can send customized response to client, which includes 
  file, plain text, html, JSON, XML etc. 
 
        response.json()        => specific JSON response 
        response.send()        => any type of response 
        response.end()        => end the response

  Updating Table Structure - SQL provides "ALTER" command to modify and update structure of table and fields. - Table structure includes adding fields, changing field name, removing field, changing data

1. Adding a new Field to table. - ALTER requires "ADD" command to add a new field. - You have to use helpers commands that specify where to add the field. - You can add "AFTER" specific field or as "FIRST" field. - Alter command can alter a table, view, constraint or any database object. - To modify the a table it require "TABLE" command. 
 
Syntax: 
   ALTER  TABLE table_name 
    ADD  field_name data_type  constraints 
   AFTER | FIRST  field_name; 
 - If you don't define the helper command then it adds new field as last field.

Syntax:

2. Modifying the field structure. - You can modify existing field structure. - It requires ALTER to use "MODIFY" command. - You can modify the data type or constraints. - You can't directly modify the foreign key constraints. - You have to define the data type of field every time while modifying the structure. 
 
Syntax: 
     ALTER  TABLE  table_name 
 
            MODIFY   field_name  data_type  constraint; 
 
Ex: 
Alter table tblproducts 
  MODIFY Qty int NOT NULL 
 - You can add or modify multiple fields with different constraints. 
 
Ex:

  MODIFY Qty int NOT NULL DEFAULT 1,

Syntax: 
 
 ALTER TABLE  table_name CHANGE COLUMN 
  old_field_name  new_name  FIRST | AFTER field_name;   // valid with beta versions 
 
Syntax:

    old_field_name   new_name data_type  FIRST|AFTER  field_name;

            hydvendors.VendorId, hydvendors.VendorName AS HydVendor

Right Join        : It returns all records of right table and null in the left table when 
                  matching records not found. 
Cross Join        : It returns null in both left and right tables when records mismatch.

                    MongoDB Validation Schema - Validation is the process of verifying user input. - Validation is required to ensure that contradictory and unauthorized data is not get stored 
into database. - MongoDB provides "jsonSchema" to configure validation for collection. - JSON Schema uses "bson" types that verify the data type of value that you insert. - It uses "validator" object for validation 
 
Syntax: 
    > db.createCollection("name", { 
            validator:{ 
            $jsonSchema: { 
                bsonType: "object", 
                required: [ key1, key2, key3, ...], 
                properties: { 
                    fieldname: { bsonType: "datatype", description:"", validator:"" } 
                } 
            } 
           } 
     }) 
 
Ex: 
db.createCollection("users", { 
     validator: { 
        $jsonSchema:{ 
            bsonType:"object", 
            required:["UserName", "Age"], 
            properties:{ 
               UserName: { 
                        bsonType:"string", 
                        description:"User Name must be a string" 
                               }, 
               Age:{ 
                        bsonType:"int", 
                        minimum:15, 
                        maximum:30, 
                        description:"age 15 to 30 only" 
                 } 
                  }         // end of properties 
            }             // end of schema 
            }                 // end of validator 
        }                // end of options 
    ) 
 - Validation Schema is defined with "validator" 
 
    > db.collection("name", { validator: { } }) 
 - Validator uses "$jsonSchema", which is object type 
 
    > db.collection("name", { validator: { $jsonSchema: { } } }) 
 - Every schema have following keys 
        a) bsonType        : It is always object type for schema 
        b) required        : It defines the fields that are required [not value] 
        c) properties        : It is an object that configures validation for every field. 
 
    > db.collection("name", { 
        validator: { 
            $jsonSchema: { 
                 bsonType: "object", 
                 required: [ "fields.." ], 
                 properties: { 
 
                  } 
             } 
         } 
        }) - Every field validation is defined as object with bsontype, validator services & description. 
 
            properties: { 
                field : {     
                       bsonType: "datatype", 
                       minimum: value, 
                       maximum: value, 
                       required: true, 
                       description: "message" 
                    }, 
                 field: {  } 
            } 
 
Note: You can add new fields that are not defined in validator. But you can't ignore 
      the fields that are defined as required fields.

    "MongoClient" class. 
 
    - Server classes are for administration 
    - Client classes are for communication 
 
Syntax: 
    const mongoClient = require("mongodb").MongoClient; 
 
 
3. MongoClient class provides a "connect()" method to connect with MongoDB server. 
 
Syntax: 
 
     mongoClient.connect("mongodb://127.0.0.1:27017") 
    .then(()=>{ 
        console.log("connected.."); 
    }) 
    .catch(()=>{ 
        console.log("Unable to connect"); 
    }) 
 
    - connect() is a JavaScript promise, which is asynchronous in request. 
    - It returns a client object on resolve. [then] 
    - It returns an error object on reject. [catch] 
 
Syntax: 
        mongoClient.connect(url).then(clientObject=>{}).catch(error=>{}).finally()

NODE EVENT 
1.What is Event? 
A. Event is a message sent by sender to its subscriber in order to notify change. 
 
2. What is event pattern? 
A. Event follows a software design pattern called "Observer". It is a communication pattern. 
Event uses delegate mechanism "function pointer". 
 
3. What is event handler? 
A. Event handler defines the action to be performed. It is a trigger for event. 
 
        onclick            : Event 
        onclick="Name()"    : Event Handler 
 
4. What is event listener? 
A. It configures event for component dynamically. 
    Listener have a trigger and action to perform. 
    You can add listener or remove listener dynamically. 
 
5. What is Event Emitter? 
A. Event Emitter is the base class for all events. 
    Every event is derived from event emitter base. 
 
Configuring Server Side Events: 
1. Import the events module 
 
     const  events = require("events"); 
 
2. Create a new object for EventEmitter class. 
 
    let  event = new EventEmitter(); 
 
3. Configure a function that works as a Subscriber, It is a  function that defines actions 
     to perform when event is triggered. 
 
    let  handleName = function() { 
 
    } 
 
 
4. Register event by using  "on()" method 
 
      event.on("eventName", handleName); 
 
      It uses addEventListener and registers a new event by specified name. 
      Every event requires a handler function. 
 
5. Event is executed by using "emit()" method. 
 
      event.emit("eventName"); 
 
 
Ex: 
const events = require("events"); 
 
let event = new events.EventEmitter(); 
 
let handlerName = function(){ 
     console.log(`Event Executed Successfully..`); 
} 
 
event.on('send', handlerName);          //onsend="handlerName" 
event.emit('send');

                                Routing - Routing enables navigation in application. - Routing is a technique used in web application to configure user and SEO friendly URL's. - Routing can be implemented client side and server side. - Server side routes are used to configure request and response. - A route can handle request from client and send a customized response. 
 
Route Flow: 
- A client request is sent to web server. - A Http Handler validates the request and sends to Routing Engine. - Routing Engine verifies the requested path by comparing with the path defined in application 
Route Table. - If requested path is matching then it send a response to client request. - If path mismatch then it sends an error page with HTTP status code. 
 
Configuring Routes: - Every route comprises of a request path and callback function. 
 
Syntax: 
        app.get("path", function() { });   // path:request path  function(){}: callback () 
 - Path comprises of entities and wild cards 
 
    "/"            It executes when user request is not for any specific. 
    "*"            It executes when requested path mismatch. [not found] 
    "/name"        It executes when requested path exactly matches with given name 
    "/colou?r"        It executes when requested path is matching regular expression 
 - The callback function comprises of various objects, the primary are 
        a) request 
        b) response 
 - A request object can request content from client, which includes 
        request.query            => query string 
        request.cookies        => cookies 
        request.body            => form body 
        request.params        => route parameters 
        etc.. 
 - A response object can send customized response to client, which includes 
  file, plain text, html, JSON, XML etc. 
 
        response.json()        => specific JSON response 
        response.send()        => any type of response 
        response.end()        => end the response 
 
Ex:

               .... 
            ]

  Express Error Handling - Errors in application are categorized into 2 types 
    a) Compile Time Errors 
    b) Run Time Errors - Compile time errors refer to syntactical errors in code. If any compile time error occurs then 
the application fails to run or start. - Run time errors occurs when compiler is unable to understand the instructions provided to 
application while using the application. - If any runtime error occurs then it leads to "Abnormal Termination" of application. - You can prevent abnormal termination using Exception handling. - The statements used for handling exceptions are 
 
        a) try        It is monitoring block, contains statements to execute. 
        b) catch        It is handler block, catches the error. 
        c) throw        It can throw exception explicitly 
        d) finally        It contains statements to execute always 
 
Error Handling in Express: - Express provides promise based functions that report errors on reject. - Reject executes when there is any issues in the request. - Express request methods provides 
    a) request 
    b) response 
    c) next - Next is derived from "NextFunction". - It provides various functionalities like differToRoute and catch the errors. - It can report the errors to express. 
 
Syntax: 
    app.get("route", (req, res, next)=>{ 
 
          // configure the actions 
          res.end(); 
          next(error); 
    }) 
 - Next function returns errors to express which you can catch using  debugger. 
 
Setup Debugger for Express: 
 - Express provides a debugger to test all the route actions defined in application. - It tracks the routes and reports the errors. 
 
Syntax: 
         >set  DEBUG=express:* & node api.js 
 
 - It will start tracking the request and shows the tracking details. - However if any issue occurred in application at runtime, it will terminate the application. - You have to make sure that you are handling exceptions using catch() for every API request. 
 
Syntax: 
   app.get("/path", (req, res, next) =>{ 
 
        mongoClient.connect("url").then(clientObject=>{ 
 
         }) 
         .catch(error=>{     
            next(error); 
         }) 
   }) 
 - Next method can send only specific error details, which include 
    a) name 
    b) message 
 
Syntax: 
        .catch(error=>{ 
            next(`${error.name} : ${error.message`); 
        })

FAQ: How we select a technology for project? 
Ans:  By verifying "AS-IS  & "TO-BE" with GAPS. 
 
     AS-IS            TO-BE            GAP 
    ------------------------------------------------------------------------------ 
    what your         what client        Java 20% 
    technology        wants?            .NET 10% 
    is capable of?                        PHP 2% 
 
 
    AS-IS === TO-BE        => Test Pass 
    AS-IS !== TO-BE        => Test Fail 
 - JavaScript based technologies use various frameworks for testing, which includes 
 
        a) Jasmine - Karma 
        b) JEST 
 - JEST is a light weight testing framework for JavaScript based applications designed in 
Angular, React, Vue, Node etc. 
 
Setup Testing Framework for your project: 
 
1. Open your project terminal and install 
 
        >npm install  jest --save-dev 
 
        
2. Your application comprises of source code and test file 
 
          home.js        => contains your logic 
 
          home.test.js    => contains test functions 
          [home.spec.js]

    <script type="module">

    </script> 
 
5. Go to your test file  "home.test.js" 
    
    - Testing every function comprises of 3 phases 
        a) Arrange 
        b) Act 
        c) Assert 
    - Arrange imports required modules. 
    - Act defines the functionality to test. 
    - Asserts reports the test results. 
    - Every testing framework have a set of various functions for every phase. 
        
        test()        : It is used to arrange and configure your test 
        expect()        : It defines the act. 
        toBe...        : These are mock functions for verifying result [assert]
